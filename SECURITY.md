# Security Policy

We take security seriously. If you find something that does not look right, a vulnerability, a leak, or just something that feels off, please tell us directly instead of posting it publicly.

## How to report

Use GitHub's **private vulnerability reporting** on this repository:
go to the Security tab and click "Report a vulnerability". This keeps
the report private until a fix is ready.

Please include:

- What you found and where
- Steps to reproduce it (a short example helps a lot)
- What you think the impact might be

That is it. No complicated forms, no bureaucracy. Just a clear description so we can look at it and fix it.

## What happens next

We will acknowledge your report within a few days, investigate it, and keep you posted on what we are doing about it. Once it is resolved, we will thank you for the report, and if you are okay with it, we can mention your contribution in the release notes.

## A note on responsible disclosure

Please give us reasonable time to fix the issue before sharing it publicly. We will do our best to act quickly.

Thank you for helping keep this project safe.
